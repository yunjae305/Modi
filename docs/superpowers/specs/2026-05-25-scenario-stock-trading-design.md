# 시나리오별 대표 종목 거래 설계

## 목표

시나리오 모드는 더 이상 지수 하나를 매수하거나 매도하는 흐름이 아니다. 사용자는 선택한 과거 시장 시나리오 안에서 대표 주식 20개를 보고, 그중 하나를 선택해 차트를 확인한 뒤 실제 주식처럼 매수와 매도를 한다.

## 범위

- 코로나 KOSPI 시나리오는 한국 대표 종목 20개를 제공한다.
- 서브프라임 S&P 500 시나리오는 미국 대형주 20개를 제공한다.
- 닷컴버블 NASDAQ 시나리오는 당시 기술주 중심 대표 종목 20개를 제공한다.
- 각 시나리오는 런타임에 외부 API를 호출하지 않고 정적 JSON 데이터를 읽는다.
- 사용자는 여러 종목을 동시에 보유할 수 있다.
- 주문은 선택된 종목의 현재 시뮬레이션 날짜 종가를 기준으로 즉시 체결된다.
- 재생 중에는 선택된 종목과 종목 목록의 가격이 실시간처럼 계속 변동된다.
- 사용자는 0.5배속, 1배속, 2배속 버튼으로 시장 시간 진행 속도를 바꿀 수 있다.
- 결과 화면은 전체 포트폴리오 기준 최종 자산, 수익률, 최대 낙폭, 거래 횟수를 계산한다.

## 데이터 모델

기존 `OHLCVBar`는 유지하고, 시나리오 종목 단위 데이터를 추가한다.

- 종목: `id`, `name`, `market`, `bars`
- 시나리오 데이터: 종목 20개 배열
- 보유 종목: 종목 ID별 수량과 평균 매수가
- 거래 기록: 날짜, 종목 ID, 종목명, 매수/매도, 수량, 가격, 금액

기존 `Scenario.dataFile`은 시나리오의 대표 지수 파일을 가리키는 역할에서, 시나리오 종목 묶음 JSON을 가리키는 역할로 바꾼다. 차트는 선택된 종목의 `bars`를 사용한다.

## 화면 구성

시뮬레이션 화면은 세 영역으로 구성한다.

- 왼쪽 또는 상단: 대표 종목 20개 목록
- 중앙: 선택 종목의 캔들 차트와 시나리오 진행 정보
- 오른쪽: 전체 자산 현황, 선택 종목 주문 패널, 시장 시간 컨트롤, 다음 날 버튼

종목 목록에는 종목명, 코드, 현재가, 전일 대비 등락률, 보유 수량을 표시한다. 종목을 클릭하면 선택 종목과 차트가 바뀐다.

## 시장 시간

시뮬레이션은 수동 진행과 자동 진행을 모두 지원한다.

- 기본 속도는 1배속이다.
- 1배속에서는 시뮬레이션 하루가 실제 5분 동안 진행된다.
- 0.5배속에서는 시뮬레이션 하루가 실제 10분 동안 진행된다.
- 2배속에서는 시뮬레이션 하루가 실제 2분 30초 동안 진행된다.
- 사용자는 재생과 일시정지를 전환할 수 있다.
- 사용자는 0.5배속, 1배속, 2배속 버튼 중 하나를 선택할 수 있다.
- 다음 날 버튼은 자동 진행과 별개로 유지한다.

하루 안의 가격은 해당 날짜의 OHLC 데이터를 기준으로 보간한다. 시뮬레이션은 하루 진행률에 따라 시가에서 시작해 고가와 저가 구간을 지나 종가에 도착한다. 종목 목록, 선택 종목 현재가, 자산 평가금액은 자동 진행 중 1초마다 갱신된다.

## 거래 흐름

매수:

- 주문 수량이 1보다 작으면 체결하지 않는다.
- 현금으로 살 수 있는 최대 수량까지만 체결한다.
- 이미 보유한 종목이면 평균 매수가를 갱신한다.

매도:

- 주문 수량이 1보다 작으면 체결하지 않는다.
- 보유 수량을 초과하면 보유 수량까지만 체결한다.
- 전량 매도하면 해당 종목의 평균 매수가는 0으로 본다.

다음 날:

- 모든 종목이 같은 `currentDay`를 공유한다.
- 포트폴리오 평가는 각 보유 종목의 현재 날짜 종가 합계로 계산한다.
- 마지막 날짜까지 진행하면 결과 화면으로 이동한다.
- 자동 진행 중 하루 진행률이 100%가 되면 다음 거래일로 넘어간다.
- 일시정지 상태에서는 가격 진행률이 멈추고 주문은 현재 표시 가격으로 체결된다.

## 오류와 경계값

- 시나리오 데이터가 비어 있거나 20개 미만이면 데이터 오류를 보여준다.
- 선택 종목 데이터가 없으면 첫 번째 종목을 선택한다.
- 종목별 거래 날짜가 다르면 가장 짧은 데이터 길이를 시뮬레이션 종료 기준으로 사용한다.
- 가격이 0 이하인 날짜에는 주문을 체결하지 않는다.
- 보유 종목이 없어도 자산 현황과 결과 화면은 정상 표시된다.
- 자동 진행 중 마지막 거래일이 끝나면 결과 화면으로 이동한다.
- 속도 변경은 현재 날짜와 진행률을 유지한 채 이후 진행 속도만 바꾼다.

## 테스트

- 시나리오 데이터 표면 테스트: 각 과거 시나리오가 대표 종목 20개 데이터 파일을 가리키는지 확인한다.
- 거래 상태 테스트: 여러 종목 매수, 일부 매도, 전량 매도, 현금 초과 매수 제한을 확인한다.
- 포트폴리오 계산 테스트: 여러 종목 보유 상태에서 날짜 이동 후 평가금액과 최대 낙폭을 확인한다.
- 시장 시간 테스트: 0.5배속, 1배속, 2배속의 하루 진행 시간이 각각 10분, 5분, 2분 30초인지 확인한다.
- 가격 보간 테스트: 하루 진행률에 따라 표시 가격이 해당 날짜의 OHLC 범위 안에서 움직이고 종가에 도착하는지 확인한다.
- 결과 계산 테스트: 전체 포트폴리오 기준 수익률과 자산 흐름이 계산되는지 확인한다.
- 빌드 검증: `node --test --experimental-strip-types tests/*.test.ts`, `npm.cmd run build`
