# Project 모디 최종 설계서

## 0. v6 최종 수정 방향

기존 v6 문서는 모의투자대회, 소셜 로그인, 랭킹, DB, 실제 시세 기반 확장을 포함하고 있었다. 최종 제출 기준에서는 이 범위를 조정한다.

기말 프로젝트 평가 기준이 React + TypeScript 프론트엔드 역량 평가이며 백엔드 서버와 DB 구현은 평가 범위에서 제외되므로, 최종 v6는 초보자 연습과 과거 시나리오 학습을 더 구체화하는 방향으로 정리한다.

| 구분 | 기존 v6 방향 | 최종 v6 방향 |
|---|---|---|
| 핵심 경험 | 모의투자대회 확장 | 초보자 투자 연습과 시나리오 학습 |
| 데이터 저장 | DB 기반 사용자, 주문, 랭킹 저장 | 브라우저 상태와 정적 JSON, 최종 가산점 범위에서 Supabase 유저 DB 포함 |
| 배포 | 프론트엔드와 백엔드 확장 가능성 병행 | Vercel 정적 배포 |
| 실제 시세 | 한국투자 Open API 기반 확장 | 최종 제출 범위 제외 |
| 경쟁 기능 | 대회방, 랭킹, 참가자 비교 | 개인 연습 결과 리포트 |
| 로그인 | 소셜 로그인 확장 | 이메일 로그인과 결과 저장만 포함 |
| 발표 초점 | 확장 아키텍처 | React 상태 변화, 타입, 컴포넌트 분리 |

## 1. 최종 서비스 정의

모디는 주식 초보자가 과거 시장을 재생하듯 체험하면서 매수, 매도, 보유 판단을 연습하는 웹서비스다.

사용자는 먼저 캔들 차트와 시장가 주문을 짧게 학습하고, 이후 실제 과거 지수 데이터를 기반으로 구성된 3개의 시나리오를 선택해 가상 자금으로 투자 판단을 연습한다.

결과 화면에서는 단순 수익률뿐 아니라 사용자의 거래 습관을 분석해 초보자가 다음 시도에서 무엇을 개선해야 하는지 알려준다.

## 2. 최종 제외 범위

| 제외 항목 | 최종 판단 |
|---|---|
| 실제 주식장 기반 모의투자대회 | 구현하지 않음 |
| 실시간 시세 API | 구현하지 않음 |
| 랭킹 시스템 | 구현하지 않음 |
| 대회방 생성 | 구현하지 않음 |
| 대규모 DB 저장 | 실시간 대회와 랭킹 저장은 구성에서 제외 |
| 회원별 포트폴리오 저장 | 실시간 포트폴리오는 제외, 결과 요약 저장만 포함 |
| 소셜 로그인 필수 플로우 | 핵심 제출 기능에서 제외 |
| 서버 주문 체결 | 브라우저 상태에서만 처리 |

제외 기능은 발표에서 범위 관리 사례로 설명할 수 있다. 프로젝트의 목적은 실제 금융 서비스가 아니라 초보자 학습용 프론트엔드 시뮬레이션이다. 단, 최종 가산점 범위에서는 이메일 로그인과 결과 저장을 위해 Supabase 유저 DB 포함 방향으로 조정한다.

## 3. 평가항목별 설계

### 3-1. 프로젝트 기획 및 설계

| 평가 질문 | 설계 답변 |
|---|---|
| 주제가 창의적인가 | 실제 투자 전 학습이라는 현실 문제를 과거 데이터 시뮬레이션으로 해결 |
| 기능 구성이 논리적인가 | 학습, 선택, 실행, 결과의 단계적 흐름 구성 |
| UI 흐름이 자연스러운가 | 초보자는 튜토리얼로, 경험자는 시나리오로 바로 이동 |
| 컴포넌트 구조가 적절한가 | 페이지, 차트, 거래 패널, 결과 카드 단위로 분리 |
| 데이터 흐름이 적절한가 | 정적 JSON 로딩 후 전역 상태로 시뮬레이션 진행 |

### 3-2. 프로젝트 구현 완성도

| 평가 질문 | 구현 기준 |
|---|---|
| 정상 실행되는가 | Vercel 배포와 `npm run build` 통과 |
| 주요 기능이 동작하는가 | 튜토리얼, 시나리오 선택, 매수, 매도, 다음 날, 결과 리포트 동작 |
| 상태가 올바르게 변경되는가 | 주문 실행 시 현금, 보유 수량, 평균 단가, 거래 기록 갱신 |
| 예외 처리가 있는가 | 잘못된 수량, 현금 부족, 보유 수량 부족, 데이터 로딩 실패 처리 |
| UI가 안정적인가 | 반응형 레이아웃과 빈 상태 메시지 제공 |

### 3-3. React + TypeScript 활용

| 평가 질문 | 구현 기준 |
|---|---|
| 컴포넌트 분리가 적절한가 | 역할별 컴포넌트로 분리 |
| props와 state가 적절한가 | 표시 데이터는 props, 시뮬레이션 상태는 store와 hook으로 관리 |
| 타입 지정이 적절한가 | 시나리오, 가격 데이터, 거래 기록, 결과 타입 정의 |
| 이벤트 타입을 올바르게 쓰는가 | 입력 변경과 버튼 클릭 핸들러를 분리 |
| React 패턴을 이해하는가 | 라우팅, 조건부 렌더링, 배열 렌더링, 커스텀 훅 사용 |

### 3-4. 발표 및 설명

| 평가 질문 | 준비 내용 |
|---|---|

| state 변경 설명 가능 | 매수와 매도, 다음 날 진행, 결과 계산 흐름 설명 |
| AI 활용 설명 가능 | 요구사항, 프롬프트, 수정 과정, 직접 검증 내용을 정리 |

## 4. 초보자 연습 모드 구체화

초보자 연습은 최종 v6에서 가장 중요한 보강 영역이다. 단순 설명이 아니라 사용자가 직접 입력하고 오류를 경험하며 배울 수 있게 설계한다.

### 4-1. 초보자 연습 목표

| 목표 | 설명 |
|---|---|
| 차트 읽기 | 캔들의 시가, 고가, 저가, 종가를 구분한다 |
| 색상 이해 | 상승 캔들과 하락 캔들의 의미를 이해한다 |
| 주문 이해 | 시장가 매수와 매도 개념을 익힌다 |
| 금액 계산 | 수량과 현재가로 주문 금액이 결정되는 것을 확인한다 |
| 오류 경험 | 잘못된 수량 입력 시 안내 메시지를 확인한다 |

### 4-2. 튜토리얼 단계

| 단계 | 화면 구성 | 사용자 행동 | 상태 변화 |
|---|---|---|---|
| 1단계 | 캔들 차트와 용어 카드 | 캔들에 마우스를 올림 | 선택한 캔들 정보 표시 |
| 2단계 | 상승, 하락 캔들 비교 | 설명 카드를 읽고 다음 클릭 | `step` 증가 |
| 3단계 | 시장가 주문 패널 | 수량 입력 | `qty` 변경 |
| 4단계 | 10주 매수 미션 | 매수 버튼 클릭 | 성공 또는 오류 메시지 변경 |
| 5단계 | 실전 진입 | 시나리오 버튼 클릭 | 시나리오 선택 화면 이동 |

### 4-3. 튜토리얼 입력 검증

| 입력 상황 | 메시지 |
|---|---|
| 빈 값 | 수량을 입력해 주세요 |
| 0 이하 | 1주 이상 입력해 주세요 |
| 10주 미만 | 이번 미션은 10주 매수입니다 |
| 10주 초과 | 너무 많이 입력했습니다. 10주만 매수해 보세요 |
| 10주 입력 | 시장가 10주 매수가 체결되었습니다 |

### 4-4. 초보자 친화 UI 문구

| 위치 | 문구 |
|---|---|
| 튜토리얼 시작 | 차트가 처음이어도 괜찮아요. 오늘은 캔들과 주문 버튼만 익히면 됩니다 |
| 캔들 설명 | 몸통은 시작가와 끝 가격, 꼬리는 하루 중 가장 높고 낮았던 가격을 보여줍니다 |
| 주문 패널 | 수량을 입력하면 예상 주문 금액이 바로 계산됩니다 |
| 오류 메시지 | 실제 투자 전에 실수해보는 것이 이 연습의 목적입니다 |
| 완료 메시지 | 이제 과거 시장 시나리오에서 직접 판단해 볼 차례입니다 |

## 7. 상태 변경 발표용 설명

### 7-1. 매수 버튼 클릭

```text
수량 입력
→ 현재가 확인
→ 주문 금액 계산
→ 현금 부족 여부 확인
→ cash 감소
→ shares 증가
→ averagePrice 재계산
→ trades에 매수 기록 추가
→ 메시지 표시
```

### 7-2. 매도 버튼 클릭

```text
수량 입력
→ 보유 수량 확인
→ 매도 금액 계산
→ cash 증가
→ shares 감소
→ trades에 매도 기록 추가
→ 메시지 표시
```

### 7-3. 다음 날 버튼 클릭

```text
currentDay 확인
→ 마지막 날이 아니면 currentDay 증가
→ 차트에 보이는 데이터 증가
→ 현재가 갱신
→ 자산 평가액 갱신
→ 마지막 날이면 결과 계산 후 결과 페이지 이동
```

### 7-4. 결과 계산

```text
최종 현재가 확인
→ 보유 주식 평가액 계산
→ 현금과 평가액 합산
→ 수익률 계산
→ 최대 손실 계산
→ 투자 성향 분류
→ 결과 리포트 표시
```

## 11. 배포 및 모바일/PWA 대응

| 항목 | 기준 |
|---|---|
| 배포 플랫폼 | Vercel |
| 실행 방식 | Vite 프론트엔드 + `/api/*` Vercel Functions |
| 빌드 명령 | `npm run build` |
| 결과 폴더 | `dist` |
| 데이터 | `public/data/` 정적 JSON |
| DB | Supabase 유저 DB 포함 |
| 서버 API | 이메일 로그인, 현재 사용자 조회, 결과 저장 |

### 11-1. Vercel 환경변수

| 환경변수 | 용도 |
|---|---|
| `VITE_API_BASE_URL=/api` | 프론트엔드 API 기본 경로 |
| `SUPABASE_URL` | Supabase 프로젝트 URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Vercel Functions 전용 DB 접근 키 |
| `JWT_SECRET` | 세션 쿠키 서명 |

`SUPABASE_SERVICE_ROLE_KEY`는 브라우저에 노출하지 않고 Vercel 서버 함수에서만 사용한다.

### 11-2. 로그인과 유저 DB

| 기능 | 설명 |
|---|---|
| 이메일 회원가입 | 이메일, 비밀번호, 닉네임을 입력해 `users`에 저장 |
| 이메일 로그인 | 비밀번호 해시 검증 후 세션 쿠키 발급 |
| 로그인 상태 확인 | `/api/auth/me`로 현재 사용자 확인 |
| 결과 저장 | 로그인 사용자의 시나리오 결과를 `result_records`에 저장 |
| 최근 결과 조회 | 최근 5개 결과를 결과 페이지에 표시 |

### 11-3. PWA 구성

| 파일 | 역할 |
|---|---|
| `public/manifest.webmanifest` | 앱 이름, 색상, 아이콘, 설치 정보 |
| `public/icons/modi-192.png` | 모바일 홈 화면 기본 아이콘 |
| `public/icons/modi-512.png` | 고해상도 앱 아이콘 |
| `public/icons/modi-maskable-512.png` | Android maskable 아이콘 |
| `public/sw.js` | 앱 shell, 오프라인 화면, 시나리오 JSON 캐시 |
| `public/offline.html` | 네트워크 실패 시 보여줄 화면 |

### 11-4. 모바일 검증 기준

| 화면 | 모바일 확인 기준 |
|---|---|
| `/` | 첫 화면 제목과 CTA가 360px 폭에서 잘리지 않음 |
| `/tutorial` | 차트와 주문 입력이 세로 흐름으로 사용 가능 |
| `/select` | 시나리오 카드가 한 열로 자연스럽게 나열 |
| `/simulation` | 차트, 자산 현황, 거래 패널이 세로로 배치 |
| `/result` | 결과 지표, 그래프, 최근 저장 결과가 모바일 폭에서 겹치지 않음 |
| `/login` | 이메일, 비밀번호, 닉네임 입력과 버튼이 터치하기 쉬움 |

Vercel 배포 후 확인할 경로는 다음과 같다.

| 경로 | 확인 내용 |
|---|---|
| `/` | 랜딩 페이지 |
| `/tutorial` | 초보자 연습 |
| `/select` | 시나리오 선택 |
| `/simulation` | 매매 시뮬레이션 |
| `/result` | 결과 리포트 |
| `/login` | 이메일 로그인과 회원가입 |
